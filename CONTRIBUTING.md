# Contributing to Invincible 🏏

First off, thank you for your interest in contributing to **Invincible – IPL Strategy Simulator**! 🎉

Whether you're fixing a bug, improving the user interface, optimizing the simulation engine, or suggesting new gameplay features, every contribution is greatly appreciated.

Our goal is to build a fun, immersive, and strategic cricket simulator that the community can enjoy and improve together.

---

# 🚀 Getting Started

## 1. Fork the Repository

Click the **Fork** button at the top of this repository to create your own copy.

---

## 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/invincible-ipl-simulator.git
```

Navigate into the project.

```bash
cd invincible-ipl-simulator
```

---

## 3. Install Dependencies

```bash
npm install
```

---

## 4. Start the Development Server

```bash
npm run dev
```

The application will start locally.

---

# 🌱 Creating a Branch

Before making changes, create a new branch.

```bash
git checkout -b feature/your-feature-name
```

Examples:

```text
feature/player-search

feature/auction-mode

feature/improved-scorecard

fix/draft-bug

fix/ui-overflow
```

---

# 💻 Coding Guidelines

Please follow these practices while contributing.

- Write clean and readable code.
- Use meaningful variable and function names.
- Keep components modular.
- Avoid unnecessary duplication.
- Follow the existing folder structure.
- Test your changes before submitting.
- Keep pull requests focused on a single feature or bug fix.

# 🧪 Testing

Before opening a Pull Request, make sure that:

- The project builds successfully.

```bash
npm run build
```

- The application runs without errors.

```bash
npm run dev
```

- Existing functionality has not been broken.

---

# 📦 Commit Messages

Use clear and descriptive commit messages.

Examples:

```text
Added auction mode

Improved match simulation

Fixed captain selection bug

Optimized draft generation

Updated Hall of Fame UI

Improved responsive layout
```

Avoid messages like:

```text
update

fix

changes

done
```

---

# 🔀 Submitting a Pull Request

Before submitting your Pull Request, ensure:

- Your code follows the project style.
- The application builds successfully.
- Screenshots are included if the UI changes.
- The purpose of the Pull Request is clearly explained.

A good Pull Request should include:

- What changed
- Why it changed
- Screenshots (if applicable)
- Testing performed

# 💡 Feature Requests

Have an idea?

# 🐞 Reporting Bugs

When reporting a bug, please include:

- Browser
- Operating System
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if possible)
